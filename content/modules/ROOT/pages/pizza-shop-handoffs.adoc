This section explains how agents hand off control to each other using LangGraph.

== Supervisor Routing

The supervisor agent routes requests to the appropriate specialist agent:

image::supervisor-routing.png[Supervisor Routing]

Routing decisions are based on:

* User intent (ordering pizza, asking about delivery, etc.)
* Current conversation state
* Agent availability and capabilities

== Wait for User Speech

The system pauses to wait for user input:

image::wait-for-user-speech.png[Wait for User Speech]

This is where Voice Activity Detection (VAD) comes into play in more advanced implementations.

== Voice Activity Detection (VAD)

VAD is mentioned here as an advanced feature. In our current implementation:

* We use manual start/stop recording
* A production system might use VAD to detect speech automatically
* VAD can improve the natural flow of conversation

== Handoff Flow

The typical handoff flow works like this:

1. User speaks → Speech converted to text (STT)
2. Supervisor receives text and analyzes intent
3. Supervisor routes to appropriate agent
4. Agent processes request and generates response
5. Response converted to speech (TTS)
6. System waits for next user input

This creates a seamless conversation experience where the user feels like they're talking to a single coherent assistant, even though multiple specialized agents may be involved.
